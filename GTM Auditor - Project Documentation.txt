1. Executive Summary
GTM Auditor is a full-stack, AI-powered web application that lets users ask natural-language questions about their Google Tag Manager (GTM) container configuration. The application ingests a raw GTM container JSON export, feeds it to Google's Gemini LLM as persistent context, and provides an interactive chat interface where users receive streamed, real-time AI analysis of their tags, triggers, variables, consent settings, and overall container health.
Core Value Proposition
ProblemSolutionGTM containers can have 100+ tags with thousands of lines of JSON — auditing them manually is slow and error-proneThe AI has the full container pre-loaded as context and can instantly answer questions about any tag, trigger, or variableContainer exports are difficult for non-technical stakeholders to understandThe LLM translates raw JSON into plain-English explanationsTypically, auditing requires exporting ? reading ? cross-referencing trigger IDs ? checking consent settings one by oneUsers simply ask "Are there any missing consent settings?" and get a structured Markdown answerHow It Works (30-Second Version)
1. A GTM container JSON is pre-loaded into Cloudflare KV storage.
2. When a user asks a question, the Cloudflare Worker fetches the container from KV, creates (or reuses) a Gemini Context Cache containing that JSON, and streams the AI response back via Server-Sent Events (SSE).
3. The React frontend renders the streamed Markdown response in real-time with a typing indicator.
4. Both the question and answer are persisted to Cloudflare D1 (SQLite) for cross-session history.

1. Send a POST with { sessionId, question }.
2. Gets a ReadableStream response body.
3. Reads chunks via reader.read() in a loop.
4. Maintains a buffer string for incomplete SSE lines.

6.6 Chat Endpoint — Detailed Walkthrough
The POST /api/chat handler is the most complex route (~120 lines). Here's the step-by-step:
1. Parse body: Extract { question, sessionId }.
2. Validate: Ensure question is non-empty, GEMINI_API_KEY exists.
3. Fetch container: KV.get('container', { cacheTtl: 3600 }) — reads from KV with a 1-hour edge cache.
4. Initialize Gemini SDK: new GoogleGenAI({ apiKey }).
5. Build conversation history from D1:
SELECT role, text FROM messages WHERE session_id = ? ORDER BY created_at ASC

Maps each row to { role, parts: [{ text }] } format, then appends the current question.
6. Get or create context cache (see §6.5).
7. Build request config:
? If cache available: { temperature: 0.3, cachedContent: cacheName }
? If cache unavailable (fallback): { temperature: 0.3, systemInstruction: SYSTEM_INSTRUCTION + containerJSON }
8. Create TransformStream for SSE output.
9. Return the readable side immediately as the HTTP response (with SSE headers).
10. Async streaming pipeline (runs after response headers are sent):
? Calls generateContentStream().
? Loops over chunks, writes data: {"text":"..."}\n\n for each.
? On stream completion, writes data: [DONE]\n\n.
? Persists to D1 via a batch of 3 statements:
1. INSERT the user's question message
2. INSERT the model's complete answer
3. UPDATE sessions SET updated_at = now
? Closes the writer.

11. Real-Time Streaming Pipeline
The streaming pipeline uses Server-Sent Events (SSE) over a Cloudflare Worker TransformStream. Here's the full end-to-end flow:


SSE Wire Format
Each SSE event is a single line:

?data: {"text":"chunk content here"}
?The stream terminator:

?data: [DONE]
?Error within the stream:

?data: {"error":"error message here"}
?Frontend Stream Parsing Logic
The sendMessageStream() method in chatApi.ts implements a robust SSE parser:
1. Buffer management: Incomplete lines are kept in a buffer variable across reader.read() calls.
2. Line splitting: buffer.split('\n') — the last (potentially incomplete) element stays in the buffer.
3. Prefix stripping: Lines not starting with data: are ignored.
4. JSON parsing: Each data: payload is JSON.parse()-d.
5. Error handling: Non-JSON parse errors (actual server errors in parsed.error) are re-thrown to the caller.
6. Termination: data: [DONE] causes the function to return { fullAnswer }.

Purpose: Strips control characters (bytes 0x00–0x1F and 0x7F–0x9F) from JSON strings, while preserving newlines (\n), carriage returns (\r), and tabs (\t).

?export function cleanGtmJsonString(originalContent: string): string {
  return originalContent.replace(/[\x00-\x1F\x7F-\x9F]/g, char => {
    if (['\n', '\r', '\t'].includes(char)) return char;
    return '';
  });
}
